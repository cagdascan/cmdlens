# cmdlens Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a TypeScript CLI that explains shell commands, generates commands from natural language via the local `codex` client, and always asks for confirmation before execution.

**Architecture:** The CLI layer parses subcommands and interactive input, then routes requests through small core services for intent detection, Codex prompt execution, safety analysis, and shell execution. Codex returns strict JSON that `cmdlens` validates before rendering or running anything, while all risky execution decisions remain local and deterministic.

**Tech Stack:** Node.js 22+, TypeScript, `commander`, `@inquirer/prompts`, `execa`, `picocolors`, `zod`, `vitest`, `tsx`

Implement a small heuristic classifier using token patterns:

- shell metacharacters
- common executable-first tokens such as `git`, `find`, `ls`, `cat`, `docker`
- sentence-like prefixes such as `show`, `find`, `delete`, `archive` followed by descriptive prose

```ts
export function detectIntent(input: string): "command" | "natural_language" {
  const text = input.trim();
  if (/[\|\>\<;&]/.test(text)) return "command";
  if (/^(git|find|ls|cat|grep|docker|npm|pnpm|yarn|rsync|tar)\b/.test(text)) return "command";
  return "natural_language";
}
```
